Postmortem: lobbyfix-webapp Deployment Failure

Date: March 8–9, 2026 Service: lobbyfix-webapp (regiztra-2-prod cluster) Severity: Deployment blocked; production service unaffected (automated rollback kept previous version live) Status: Resolved

Summary

A deployment pushed on March 8 failed to start correctly and triggered an automated rollback. The root cause was a memory configuration mismatch: the application was configured to use 768 MB of memory, but the hosting container was limited to 512 MB. Every new deployment attempt was terminated by the system before it could pass health checks. The issue was resolved on March 9 by increasing the container memory limit to 1024 MB.

Root Cause

The application server is configured to reserve 768 MB for its Node.js memory heap. The ECS task (the containerized unit that runs the app) had a hard memory ceiling of 512 MB — less than what the application expected. When the new version started, the operating system terminated the process before it could fully initialize and serve traffic.

The previous version was unaffected because it had been deployed before this constraint became a problem. The new image had a slightly larger memory footprint at startup, which pushed it over the limit.

Resolution

The container memory limit was raised from 512 MB to 1024 MB in the task definition. The new deployment completed successfully:

  • Startup time: ~11 seconds (vs. ~20s on the failed attempts)
  • Health check: passed
  • Status: Running and healthy

Prevention

The following measures have been identified to prevent recurrence:

  1. Memory headroom policy — container memory limits should always exceed the Node.js heap size by at least 256 MB to account for the OS, web server, and other processes running alongside the app.

  2. Pre-deployment local testing with resource constraints — running the Docker image with the same memory limits as production (--memory=512m) before deploying would have surfaced this issue immediately.

  3. Deployment circuit breaker — enabling ECS's automatic circuit breaker with rollback would have reduced the failure window from 5 hours to the first failed health check cycle (~2–3 minutes).
